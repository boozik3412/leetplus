# LeetPlus: саммари для нового чата

Дата: 14.07.2026

## Проект и рабочий процесс

- Репозиторий: `C:\Users\ALIENWARE\Desktop\leetplus`.
- Ветка production: `main`, GitHub: `https://github.com/boozik3412/leetplus`.
- Прод: `https://leetplus.ru`, API: `https://api.leetplus.ru`.
- Продакшен автоматически деплоит `origin/main`. API и web на VDS собираются последовательно; параллельная сборка может привести к OOM и битому `.next`.
- Пользователь просит проверять итог прежде всего на production. Не перезапускать локальные сервисы и не обновлять локальную БД без прямого запроса.
- Не включать в сообщения, коммиты и документацию пароли, токены, raw phone или payload Langame.

## Стек и ключевые маршруты

- Monorepo `pnpm`: Next.js 16 / React 19, NestJS 11, PostgreSQL / Prisma.
- Админский модуль: `/guests/gamification`.
- Гостевой игровой модуль: `/game`, совместимый URL `/play/game`.
- Гостевой вход: `/game/auth`; выбор клуба: `/game/clubs`; награды: `/game/rewards`.
- Главная API-модель игры: `GET /guest-portal/session/game-summary`.

## Текущий игровой контракт

- Расширенный редактор управляет условиями, статусом и клубами. После подтверждения активации элемент сразу участвует в игровом контуре выбранных клубов. Визуальный редактор публикует только layout/appearance.
- Battle Pass и задания: награды создаются и отправляются автоматически по факту выполнения условия. Гость не должен нажимать Claim.
- Лутбокс: pipeline может только создать entitlement (право открыть). Награда и случайный приз появляются только при ручном открытии гостем. Разблокированный лутбокс не должен исчезать после окончания временного окна/сессии.
- Чекин отделен от заданий; один раз за календарный день в одном клубе по timezone клуба. Успех, повтор и ошибка показываются модалками.
- Тип сессии: только `HOURLY` и `PACKAGE_OR_SUBSCRIPTION` (пакет либо абонемент). Langame не дает надежный структурированный маркер конкретного названия пакета, поэтому для диагностики есть второй слой guest logs / activity ledger.

## Live-пайплайн и UX

- `POST /guest-portal/session/app-open` обрабатывает активные `APP_OPEN` правила и возвращает `previousSummary`; для него отключена авто-выдача лутбоксов.
- `GuestGamificationPipelineSchedulerService` работает внутри API раз в 15 секунд (default), обрабатывает подготовленные snapshot-факты и автоматически создает reward для Battle Pass/заданий.
- В production scheduler включается при `SYNC_SERVICE_TOKEN`, если `GUEST_GAME_PIPELINE_SCHEDULER_ENABLED` не задает явное значение.
- `/game` опрашивает summary раз в 15 секунд, сравнивает предыдущее/новое состояние и показывает новые награды в последовательной очереди модалок. После закрытия одной открывается следующая.
- Последний коммит, который добавил auto-reward pipeline: `46fd619 Automate battle pass reward progression`.

## Диагностика и важные проверки

- На `/guests/gamification` доступны решения rules, facts игрового журнала, freshness и reward history. Это основной источник для расследования, почему условие/награда не сработали.
- В проверке Battle Pass смотрите: активность правила, trigger, матч факта Langame, auto-reward/ledger, новый current step в game-summary.
- В проверке лутбокса отличайте `доступен к открытию` от `награда выдана`: лутбокс никогда не обязан выдаваться в фоне.
- Черновик не должен участвовать в боевой обработке и блокировать активное правило.
