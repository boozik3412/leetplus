# Геймификация гостей LeetPlus

Дата: 03.06.2026
Статус: рабочий guide для реализации Guest Game Hub.

## 1. Назначение

Геймификация гостей - отдельный игровой слой LeetPlus для повторных визитов, тихих часов, бара, турниров и удержания. Модуль должен вести пользователя от общего сценария к конкретной настройке:

`игровой профиль -> событие гостя -> правило -> награда -> ручная выдача -> эффект`

Модуль не должен превращаться в набор промо-акций внутри маркетинга. Маркетинг отвечает за кампании и промо-эффект, а Guest Game Hub отвечает за игровые профили, XP, уровни, лутбоксы, миссии, battle pass и кошелек наград.

## 2. Где находится

- Web route: `/guests/gamification`
- Sidebar: `Гости -> Геймификация`
- Frontend: `apps/web/src/app/(app)/guests/gamification/page.tsx`
- UI component: `apps/web/src/components/guest-gamification-panel.tsx`
- API: `/guests/gamification/*`
- Backend: `apps/api/src/guest-gamification/*`
- Data models: `GuestGameProfile`, `GuestGameLootBox`, `GuestGameMission`, `GuestGameSeason`, `GuestGameReward`, `GuestGameEvent`

## 3. Что уже реализовано

- Игровые профили гостей: связь с Guest Profile или CRM-лидом, display name, контакт, Telegram/MAX identity, XP, уровень и статус.
- Лутбоксы: статус, триггер, аудитория, сегмент, тип сессии, клубы, период, лимиты, вероятности, бюджет, антифрод и ручное согласование.
- Миссии: тип, триггер, XP, прогресс, период, клубы, аудитория, бюджет, лимиты, условия, антифрод и ручное согласование.
- Battle Pass: сезон, период, XP-правила, уровни, free/premium rewards, premium mode, бюджет и ручное согласование.
- Кошелек наград: статус `PENDING/APPROVED/PAID/CANCELED/EXPIRED`, код, срок, связка с профилем/гостем/лутбоксом/миссией/сезоном/клубом.
- Игровые события: ручная фиксация событий и XP-дельты для профиля.

## 4. Ограничения v1

- Автоматическая запись наград, бонусов или промокодов в Langame выключена.
- Награды выдаются через ручную очередь, код, кассира или внутреннее подтверждение LeetPlus.
- Рабочие страницы не должны делать живые запросы к Langame при открытии; нужны подготовленные snapshot-факты.
- Telegram/MAX используются как будущий игровой интерфейс только после согласий, отписки и юридической проверки.
- Телефон, email, ФИО, Telegram/MAX identity и коды наград считаются чувствительными данными.
- Событие `старт сессии` пока не считается realtime-гарантией, пока не подтвержден источник свежих событий.

## 5. Ближайший roadmap

1. Сделать обзорный UX-слой Guest Game Hub: сценарные входы в профили, лутбоксы, миссии, battle pass и кошелек.
2. Упростить конструкторы: заменить часть JSON на бизнес-контролы для клуба, аудитории, типа сессии, тихих часов, лимитов и бюджета.
3. Добавить тестовый запуск правила без реальной выдачи награды.
4. Подключить подготовленные Langame snapshot-факты для eligibility и прогресса.
5. Развести права: просмотр геймификации, управление правилами, согласование наград, просмотр чувствительных данных.
6. Подготовить Telegram bot/Mini App как первый гостевой интерфейс; MAX подключать вторым этапом.

## 6. Критерии безопасного запуска

- Любая награда имеет бюджет, лимит, срок, статус и аудит.
- Пользователь видит, почему правило сработало или не сработало.
- Одна и та же награда не может быть выдана повторно из-за повторной синхронизации.
- Автоматическая выдача в Langame не включается без подтвержденного write API, идемпотентности, ролей, аудита и rollback-сценария.
- Guest Game Hub раскрывает информацию от общего к частному: сначала состояние игровой экономики, затем конкретные правила и ручная выдача.
